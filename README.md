# Cracked.sh Orchestrator

This project orchestrates the scraping of combolists from cracked.sh MyBB forum. It uses a Scrapy spider to identify top combolist threads and then employs `Camoufox` (a browser automation tool) to reveal hidden content and extract links from these threads. The extracted links are intended for further processing (e.g., with CyberDropDownloader).

## Features

- Scrapes top combolist threads from cracked.sh using Scrapy.
- Identifies threads posted "today" based on relative time indicators.
- Extracts numerical values from thread titles to prioritize relevant content.
- Uses `Camoufox` to interact with forum pages, post replies (e.g., "thx") to reveal hidden content.
- Extracts direct links from the revealed hidden content.
- Saves extracted links to a file for external tools.

## Prerequisites

- Python 3.x
- pip
- Docker (for Camoufox, or ensure compatible browser environment)
- Scrapy
- Camoufox library (ensure it's installed and configured correctly, including its dependencies)

## Installation

1.  Clone the repository:
    ```bash
    git clone https://github.com/your-username/crackedsh_orchestrator.git
    cd crackedsh_orchestrator
    ```
2.  Install Python dependencies:
    ```bash
    pip install -r requirements.txt # (Assuming a requirements.txt exists or creating one)
    ```
3.  Ensure Camoufox is set up and accessible. This might involve Docker for a headless browser environment.

## Usage

To run the orchestrator:

```bash
python main_orchestrator.py
```

This will:
1.  Run the Scrapy spider to fetch top combolist threads.
2.  Process these threads using Camoufox to reveal hidden content and extract links.
3.  Save the extracted links to `crackedsh_extracted_links.txt`.

## Project Structure

- `main_orchestrator.py`: The main script that orchestrates the Scrapy spider and the Camoufox-based content revealer.
- `crackedsh_scraper_mybb/`: Contains the Scrapy project for scraping the MyBB forum.
    - `crackedsh_scraper_mybb/scrapy_mybb_scraper/`: Scrapy spider, items, and pipelines.
        - `spiders/mybb_spider.py`: The Scrapy spider definition.
        - `pipelines.py`: Processes scraped items, sorts them, and saves results.
        - `settings.py`: Scrapy project settings.
    - `mybb_scraper_scrapy.py`: An alternative direct entry point for running the Scrapy spider (redundant with `run_scraper.py` within the Scrapy project, and `main_orchestrator.py` is the primary orchestrator).
- `scrapers/`: Contains general scraping utility scripts.
    - `forum_scraper.py`: Implements the `Camoufox`-based logic to reveal hidden content and extract links from forum threads.
- `sample_urls.txt`: A sample file containing URLs to be processed.
- `top_combolists_today.json`: Output file from the Scrapy spider containing raw scraped thread data.
- `crackedsh_extracted_links.txt`: Output file containing all links extracted from hidden content.
- `post_state.json`: (Generated by `forum_scraper.py`) Tracks daily post count for rate limiting.
- `profile/`: (Generated by `forum_scraper.py`) User data directory for `Camoufox` for persistent browser context.

## Output Files

- `top_combolists_today.json`: JSON file containing details of the top combolist threads identified by the Scrapy spider.
- `crackedsh_extracted_links.txt`: A plain text file, each line containing a URL extracted from the hidden content of the processed threads.
- `post_state.json`: Internal state file for rate limiting the Camoufox operations.
